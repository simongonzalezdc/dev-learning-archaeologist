# Signal Map

Use this map to recognize what kind of coaching the user needs.

| User signal | Likely friction | Coach response |
| --- | --- | --- |
| "I cannot start" | Activation friction | Five-Minute Runway |
| "I do not know what to do first" | Prioritization overload | Choose-One Protocol |
| "I have so much to do" | Working-memory overload | Brain Dump To One Action |
| "I disappeared" | Shame plus context loss | Return Without Shame |
| "I am behind" | Time threat plus shame | State check, then deadline triage |
| "This should be easy" | Moralizing the blocker | Reframe as friction, reduce task size |
| "I need to finish before I stop" | Hyperfocus risk | Hyperfocus Exit |
| "The review made me feel awful" | Feedback threat | Code Review Armor |
| "I keep opening other tabs" | Stimulation seeking or avoidance | Start In The Middle, reduce friction |
| "I have no idea where I left off" | Context loss | Artifact reconstruction |
| "Everything is urgent" | Threat compression | Safety/deadline/blocker ordering |
| "I am too tired to think" | Low capacity | Biological check, minimum viable move |

## Friction Types

### Activation Friction

The user knows the task but cannot begin.

Best interventions:

- First contact.
- Five-minute timer.
- Body double.
- Start in the middle.

### Working-Memory Overload

The user is carrying too many open loops internally.

Best interventions:

- Brain dump.
- Sort and park.
- Return one action.

### Feedback Threat

The user treats critique as danger or identity evidence.

Best interventions:

- Sort comments.
- Identify behavior-changing items.
- Delay meaning-making.

### Time Blindness

The user underestimates setup, transitions, emotional resistance, or recovery.

Best interventions:

- 1.5x to 3x buffer.
- Backward planning.
- Visible time box.
- Transition alarms.

### Hyperfocus Hangover

The user spent too long in deep work and now feels depleted or disoriented.

Best interventions:

- Biological reset.
- Breadcrumb.
- No extra work extraction.
- Recovery window.

### Shame Re-Entry

The user returns after delay and expects punishment.

Best interventions:

- No penalty language.
- Artifact reconstruction.
- One visible re-entry move.

## Language To Use

- "No penalty."
- "We reconstruct from artifacts."
- "That is friction, not failure."
- "I am holding the rest."
- "One visible move."
- "Done enough to park."

## Language To Avoid

- "Just do it."
- "You should have..."
- "Why did you wait?"
- "Be disciplined."
- "This is easy."
- "You are overthinking."
- "You need to get organized."
